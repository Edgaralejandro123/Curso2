Creando mi commit

Pasos a seguir:

  1. Paso 1: git init
  2. ls -a
  3. git status
  4. git add
  5. git commit -m "pagina.html modificada"
  6. git log

git init: comando para inicializar un repositorio local. (master). comando: ls -a ; mostrar archivos ocultos.

git status: ofrece una descripción del estado de los archivos.

git add: comando para que nuestro repositorio sepa de la existencia de un archivo o de sus últimos cambios, no almacena las actualizaciones de forma definitiva, únicamente las guarda en algo que conocemos como “Staging Area” (área de montaje o ensayo). Se puede armar como git add ArchivoEjemplo.js (solo agrega ese archivo) o git add (agrega todos los archivos modificados de la carpeta donde estas con el punto.)

commit -m " mensaje " : comando para almacenar definitivamente todos los cambios que por ahora viven en el área de preparación. En el mensaje ponemos el mensaje que explica los cambios cometidos.

git log: revisar cuantos y cuales commits, mostrara el codigo del commit y el nombre que se le dio

git push : comando para mandar nuestros commits a un servidor remoto. git status : ofrece una descripción del estado de los archivos. git pull : sirve para recibir cambios de repositorio remoto a local.
